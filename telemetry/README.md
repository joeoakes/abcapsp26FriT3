# 📡 Secure Telemetry & Robot Control

## 📌 Telemetry Overview
The Mini-Pupper swarm communicates using secure HTTPS-based JSON telemetry.

Telemetry is generated by the Maze App and transmitted over TLS-encrypted channels to:
* **The Mini-Pupper robot** (for movement execution)
* **The Logging Server** (MongoDB telemetry persistence)
* **The AI Server** (Redis mission storage and vector processing)

All telemetry communications operate on assigned secure team ports and use self-signed or certificate-based authentication depending on deployment mode.

---

## 🔐 Communication Protocol
* **Protocol:** HTTPS (TLS 1.2+)
* **Method:** POST
* **Content-Type:** `application/json`
* **Assigned Port (Friday Team 3):** `8449`
* **Endpoint:** `/move`
* **Security:** Self-signed certificate testing supported (`-k` flag)

This ensures encrypted, authenticated robot command transmission.

---

## 📦 Telemetry Payload Schema
Each movement event from the Maze App generates a structured JSON payload:

```json
{
  "event_type": "player_move",
  "input": {
    "device": "joystick",
    "move_sequence": 1
  },
  "player": {
    "position": {
      "x": 0,
      "y": 1
    }
  },
  "goal_reached": false,
  "timestamp": "2026-02-10T20:49:22Z",
  "move_dir": "forward"
}
```
## 🧩 Payload Field Definitions

| Field | Description |
| :--- | :--- |
| `event_type` | Type of telemetry event generated |
| `input.device` | Movement source (keyboard / joystick) |
| `input.move_sequence` | Incremental movement counter |
| `player.position` | Current maze coordinates |
| `goal_reached` | Mission completion flag |
| `timestamp` | ISO 8601 UTC timestamp |
| `move_dir` | Movement directive for robot |
